Extracción desde bases de datos relacionales

Las bases de datos relacionales siguen siendo una de las principales fuentes de datos en BI. En este contexto, la extracción se realiza mediante consultas estructuradas que permiten recuperar información de tablas y vistas.

Existen dos enfoques principales:

  • Extracción completa: se recupera la totalidad de los datos en cada ejecución.
  • Extracción incremental: solo se extraen los datos que han cambiado desde la última ejecución.

La extracción incremental es ampliamente utilizada en entornos productivos, ya que reduce el volumen de datos procesados y mejora el rendimiento del sistema.

Extracción desde bases de datos NoSQL

Con el crecimiento del Big Data, han surgido bases de datos NoSQL diseñadas para manejar grandes volúmenes de información no estructurada o semiestructurada. Estas bases incluyen tecnologías como MongoDB, Cassandra o Redis.

La extracción desde estos sistemas presenta características particulares:

  • Estructuras de datos flexibles (JSON, documentos).
  • Ausencia de esquemas rígidos.
  • Alto volumen y velocidad de datos.

En este caso, los procesos ETL deben adaptarse a formatos menos estructurados, lo que implica un mayor esfuerzo en la fase de transformación (Provost & Fawcett, 2013).

Extracción mediante APIs (Application Programming Interfaces)

Las APIs se han convertido en una fuente clave de datos en BI, especialmente en entornos digitales donde las aplicaciones intercambian información constantemente.

La extracción mediante APIs permite acceder a datos en tiempo real o bajo demanda, lo que resulta especialmente útil para:

  • Integración con plataformas externas.
  • Análisis de datos en línea.
  • Consumo de servicios web.

En este contexto, los datos suelen recibirse en formatos como JSON o XML, lo que requiere procesos adicionales de transformación para integrarlos en estructuras analíticas.
